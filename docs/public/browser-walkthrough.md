# RDLeader Browser Walkthrough over Public Demo State

> Public-safe browser walkthrough for RDLeader using the deterministic fake-data database generated by `pnpm demo:reset`.

## What this walkthrough proves

This walkthrough connects the public proof surfaces into one browser path:

1. reset deterministic fake demo state;
2. start the server against that fake database without default/private seed data;
3. open the web UI;
4. inspect overview, runtime dispatch/result, approval request, QA evidence, and onboarding surfaces;
5. keep every visible value public-safe.

## Setup commands

Terminal 1:

```bash
pnpm demo:reset
pnpm demo:server
```

Terminal 2:

```bash
pnpm dev:web
```

Then open:

```text
http://127.0.0.1:5173
```

`pnpm demo:server` is a convenience wrapper around:

```bash
RDLEADER_DATABASE_URL=../../data/public-demo.db \
RDLEADER_SEED_MODE=none \
RDLEADER_AUTONOMY_SCHEDULER=false \
pnpm dev:server
```

The `../../data/public-demo.db` path is relative to the server package because `pnpm --filter @rdleader/server dev` runs from `apps/server`. The important setting is `RDLEADER_SEED_MODE=none`: it prevents the normal local seed employees from being inserted into the public demo database.

## Expected fake workers

The sidebar should show only synthetic workers from the public demo reset path:

- `Alex Runtime`
- `Maya Systems`

If a screenshot shows real people, private project names, local home-directory paths, app IDs, open IDs, chat IDs, message IDs, QR artifacts, or internal document links, do not publish it.

## Browser QA checklist

Use this minimal checklist before publishing screenshots or a recording:

- [ ] `pnpm docs:check` passes.
- [ ] `pnpm test` passes when code changed.
- [ ] `pnpm demo:reset` prints 2 employees, 4 work items, 2 runtime dispatches, 2 runtime results, 2 approval requests, and 2 QA episodes.
- [ ] `/employees` from the demo server returns only `alex-runtime` and `maya-systems`.
- [ ] The web UI renders `Alex Runtime` and `Maya Systems` without showing the default local seed employees.
- [ ] Screenshots avoid browser address bars if they include local filesystem or private host details.
- [ ] The recording does not show live integration login state, QR onboarding, or raw terminal output.

curl -fsS http://127.0.0.1:3001/employees/alex-runtime/approval-requests
