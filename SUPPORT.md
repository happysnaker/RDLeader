# Support RDLeader

RDLeader is a local-first control-plane experiment for supervising multiple AI R&D workers. The public repository is being released in safe slices: architecture, QA evidence, runtime endurance notes, demo material, and sponsorable packaging work.

## Fastest support path

- Support page: <https://happysnaker.github.io/support/>
- Best payment note: `RDLeader #issue`, for example `RDLeader #2`

## Current public sponsorware targets

| Issue | Outcome | Why it matters |
|---|---|---|
| [#2 Public-safe demo walkthrough](https://github.com/happysnaker/RDLeader/issues/2) | fake-data screenshots or a short demo video | helps people understand the agent-ops control plane quickly |
| [#1 DevPlan bundle sanitization](https://github.com/happysnaker/RDLeader/issues/1) | split local DevPlan features into public-safe docs and commits | turns private local proof into reusable public artifacts |

## What sponsorship funds

Sponsorship funds public work by default:

- sanitized docs and diagrams
- fake-data demo assets
- runtime endurance deep dives
- public QA evidence summaries
- README / project-page packaging
- reviewable public commits that avoid private identifiers and raw local logs

## What will not be published

Public work must not include:

- app secrets or credentials
- real chat / open / message identifiers
- QR onboarding images
- private workspace paths
- internal document URLs
- raw live integration logs
- payment screenshots or personal account identifiers
